Deployment Flow

This page outlines the steps involved in the deployment process.

  1. Write code
  2. Commit changes
  3. Push code to GitHub
  4. Jenkins triggers build
  5. Build Docker images
  6. Push to DockerHub
  7. Apply Kubernetes deployment
    • Pulls the images: When you apply your deployment.yaml (or Helm chart, etc.), Kubernetes worker nodes pull the Docker images from DockerHub.
    • Creates Pods: A Pod is the smallest deployable unit in k8s (usually runs one container, sometimes multiple tightly coupled ones).
    • Maintains desired state: If you say replicas: 3, k8s ensures 3 Pods are always running. If one crashes, it replaces it automatically.
    • Networking & Services: k8s services (ClusterIP, NodePort, LoadBalancer) handle communication between Pods and expose your app outside the cluster. In your flow, the LoadBalancer service makes your app accessible to the internet.
    • Scaling: You can scale up/down with a single command (kubectl scale or via HPA).
    • Monitoring & Healing: k8s continuously monitors Pods, restarts failed ones, and can reschedule them on different nodes if hardware fails.
  8. LoadBalancer exposes your app to the world
  9. Scalable, monitored, cloud-hosted MERN app
