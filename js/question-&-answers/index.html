Deep Copy & Shallow copy
Shallow Copy

A shallow copy of an object/array means:

  • The top-level properties are copied.
  • But if a property is itself an object/array, the reference is copied (not the actual data).
  • So nested objects are shared between the original and the copy.

Example

const obj1 = {
    name: "Alice",
    address: {
        city: "Delhi",
        pin: 110001
    }
};

const shallowCopy = { ...obj1 }
            OR
const shallowCopy = obj1
            OR
const shallowCopy = Object.assign({}, obj1)

shallowCopy.name = "Bob"; // changes only obj2

shallowCopy.address.city = "Mumbai"; // ‚ö† affects obj1 also!

console.log(obj1);
// { name: "Alice", address: { city: "Mumbai", pin: 110001 } }

console.log(shallowCopy);
// { name: "Bob", address: { city: "Mumbai", pin: 110001 } }


Deep Copy

  • When we assign an object to another object, to create a deep copy we use JSON.Parse and JSON.Stringify OR structuredClone so that change in any one object may not affect the other object as well
  • This method is useful when the object is small and has serializable properties. But if the object is very large and contains certain non-serializable properties then there is a risk of data loss

Example

const original = {
    name: "Varun",
    address: {
        city: "Delhi"
    }
};

const deepCopy = JSON.parse(JSON.stringify(original));
                OR
const deepCopy = structuredClone(original); // deep copy

deepCopy.address.city = "Mumbai";

console.log(original.address.city); // ‚úÖ "Delhi" ‚Äì original not affected

üîë Final Rule:

  • Primitives (string, number, boolean, null, undefined, symbol, bigint) ‚Üí always copied by value (safe in both shallow & deep).
  • Objects/arrays/functions ‚Üí copied by reference in shallow copy, cloned in deep copy.
Difference between forEach, for..of and for..in loops
forEach

A method available on arrays. It executes a callback function once for each element.

  • ‚úÖ Best for: Iterating arrays when you want to run a function on each element.
  • ‚ùå Cannot break/return early (unlike for or for...of).

Example

const arr = [10, 20, 30];

arr.forEach((value, index) => {
console.log(index, value);
});

// Output:
// 0 10
// 1 20
// 2 30

for...of

Iterates over iterable objects (like arrays, strings, Sets, Maps). It gives you the values directly.

  • ‚úÖ Clean way to loop through array values.
  • ‚úÖ Works on all iterables, not just arrays.
  • ‚ùå Doesn‚Äôt give you the index (but you can use .entries() if needed).

Example

(Array):
const arr = [10, 20, 30];

for (const value of arr) {
  console.log(value);
}

// Output:
// 10
// 20
// 30



(String):
for (const char of "Hi") {
  console.log(char);
}

// Output:
// H
// i

for...in

Iterates over enumerable property keys of an object (including inherited ones).It gives you the keys/indexes, not the values.

  • ‚úÖ Best for objects (to get property names).
  • ‚ùå Not recommended for arrays (order not guaranteed, might include prototype properties).

Example

(Object):
const obj = { a: 1, b: 2, c: 3 };

for (const key in obj) {
  console.log(key, obj[key]);
}

// Output:
// a 1
// b 2
// c 3
Promises
It is an object which represents the eventual completion (or failure) of an asynchronous operation and its resulting value.

Example

Promises in Sequence
Promises in sequence means ensuring that each asynchronous operation starts only after the previous one has completed

Example

const urls = ['url1', 'url2', 'url3'];

function fetchData(url) {
    return new Promise(resolve => {
        setTimeout(() => resolve(`Fetched from ${url}`), 1000);
    });
}

urls.reduce((promiseChain, currentUrl) => {
    return promiseChain.then(() => fetchData(currentUrl).then(console.log));
}, Promise.resolve());


‚úÖ Using async/await in a loop (Recommended for readability)
async function fetchInSequence() {
    for (let url of urls) {
        const data = await fetchData(url);
        console.log(data);
    }
}

NOTE:

  • map + Promise.all() is for parallel execution.
  • forEach + async/await won‚Äôt work for sequencing because forEach doesn‚Äôt wait.
  • Use for...of or reduce() to control sequence.

Promise.all(), Promise.allSettled(), Promise.race(), Promise.any()

Promise.all()

  • Waits for all promises to resolve
  • If any one fails, the entire result is rejected

Example

                    const p1 = Promise.resolve(1);
                    const p2 = Promise.resolve(2);
                    const p3 = Promise.reject("Error");

                    Promise.all([p1, p2, p3])
                    .then(results => console.log(results))
                    .catch(err => console.error("Caught:", err));

                    ‚ùå Output: Caught: Error

Promise.allSettled()

  • Waits for all promises to settle (resolve or reject)
  • Never rejects. Returns an array of { status, value/reason }.

Example

                    const p1 = Promise.resolve("A");
                    const p2 = Promise.reject("Failed");
                    const p3 = Promise.resolve("B");

                    Promise.allSettled([p1, p2, p3])
                            .then(results => console.log(results));

                    ‚úÖ Output:
                    [
                        { status: 'fulfilled', value: 'A' },
                        { status: 'rejected', reason: 'Failed' },
                        { status: 'fulfilled', value: 'B' }
                    ]

Promise.race()

  • Returns the first settled promise (resolved or rejected)

Example

                    const p1 = new Promise(resolve => setTimeout(() => resolve("A"), 300));
                    const p2 = new Promise((_, reject) => setTimeout(() => reject("B"), 200));

                    Promise.race([p1, p2])
                    .then(result => console.log("Resolved:", result))
                    .catch(err => console.log("Rejected:", err));

                    ‚ùå Output: Rejected: B (p2 settled first)

Promise.any()

  • Returns the first fulfilled promise
  • Ignores rejections unless all fail, then throws AggregateError.

Example

                    const p1 = Promise.reject("Fail 1");
                    const p2 = Promise.reject("Fail 2");
                    const p3 = Promise.resolve("Success!");

                    Promise.any([p1, p2, p3])
                    .then(result => console.log("Success:", result))
                    .catch(err => console.log("All failed:", err));
                    
                    ‚úÖ Output: Success: Success!

Event Loop
  • During execution of code, if the V8 engine encounters any web api's then those stack of code are kept outside global execution context of call stack
  • Once the global execution context is finished and all code are executed, the code kept outside are pushed into the CALLBACK QUEUE
  • Now, the event loop checks that if anything operation is pending or waiting in callback queue to be executed
  • If the event loop finds any operation, it extracts from callback queue and puts in the call stack to be executed
  • So, in simple terms event loop is a process to check & compare if there is any operation pending in the callback queue only after the call stack is empty. If it is then it will move the said operation from call back queue to call stack

Detailed Explanation

Foundation

JavaScript is single-threaded with a call stack, but handles async operations through the event loop coordinating between the JS engine and runtime APIs.

Components
  • Call Stack: Executes code synchronously
  • Web APIs: Handle setTimeout, DOM events, HTTP requests
  • Event Loop: The scheduler that pulls the next callback to run.
  • Queues (multiple):
    1. Task / Macrotask Queue -> lower priority, bigger chunks.
    2. Microtask Queue -> higher priority, tiny follow-ups.
    3. Rendering & rAF (browser) -> animation frame callbacks.
    4. Idle Callbacks (browser) -> requestIdleCallback.
    5. Node-specific -> process.nextTick queue (even higher than microtasks), plus Node‚Äôs phase queues.

Async operations go to Web APIs, then callbacks queue up. The event loop moves queued tasks to the call stack only when it's empty.

Key Insight

Microtasks always run before regular tasks. This is why Promise.then() executes before setTimeout(0) - microtasks have priority.

Follow-up Ready

What if sync code blocks? - It freezes the entire event loop and UI since there's only one thread. Break heavy work into chunks or use Web Workers.

generator function in js
  • A generator function allows you to pause and resume function execution using the yield keyword
  • Calling a generator function does not run it immediately
  • It returns an iterator object, and you use .next() to execute step by step
  • Each time you call .next(), it runs the function until it hits a yield statement, returning the yielded value
  • When the function completes, .next() returns { value: undefined, done: true }

Example

                    // Generator function in javascript
                    function* greetGenerator() {
                        yield "Hello";
                        yield "Hi";
                        yield "Welcome";
                    }

                    const greeter = greetGenerator();

                    console.log(greeter.next()); // { value: "Hello", done: false }
                    console.log(greeter.next()); // { value: "Hi", done: false }
                    console.log(greeter.next()); // { value: "Welcome", done: false }
                    console.log(greeter.next()); // { value: undefined, done: true }

Pipe & Compose

Pipe and Compose are functional programming utilities that let you combine multiple functions into a single function. The only difference is order of execution:
pipe: left-to-right
compose: right-to-left

Example

const add = x => x + 1;
const multiply = x => x * 2;
const subtract = x => x - 3;

// Using pipe
const pipe = (...fns) => input => fns.reduce((acc, fn) => fn(acc), input);
const pipeResult = pipe(add, multiply, subtract)(5); // ((5 + 1) * 2) - 3 = 9

// Using compose
const compose = (...fns) => input => fns.reduceRight((acc, fn) => fn(acc), input);
const composeResult = compose(add, multiply, subtract)(5); // (5 - 3) * 2 + 1 = 9

// Simple explanation:
const compose = (...fns) => {
    return function(input) {
        return fns.reduce(function(acc, fn) {
            return fn(acc); // apply current function to result of previous
        }, input) // start with initial input
    }
}
